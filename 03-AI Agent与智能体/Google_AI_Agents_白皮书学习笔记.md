# 第一部分：AI Agent 核心概念

## 1.1 从预测性AI到自主智能体

人工智能正在发生范式转变。多年来，人们关注的是擅长处理被动、离散任务的模型：回答问题、翻译文本或根据提示生成图像。这种范式虽然强大，但需要人类在每个步骤中持续指导。

如今我们正见证范式转变——人工智能正从单纯预测或生成内容，迈向能够自主解决问题并执行任务的新型软件类别。

### 关键定义

AI Agent（自主智能体）：由模型、工具、协调层和部署组合而成，通过循环调用语言模型来实现目标的完整应用程序。它不是静态工作流中的简单AI模型，而是具备完整功能的应用程序，能够制定计划并采取行动以达成目标。

## 1.2 智能体问题解决过程（五步循环）

Agent 通过持续循环的过程来实现其目标，可分解为五个基本步骤：

- 获取任务：流程由特定的高层级目标启动（用户提供或自动化触发器启动）

- 扫描场景：Agent 感知环境以获取上下文信息（用户请求、短期记忆、可用工具等）

- 深度思考：推理模型驱动的核心"思考"循环，分析任务与场景并制定计划

- 执行行动：协调层执行计划的首个具体步骤，选择并调用相应工具

- 观察与迭代：Agent 观察行动结果，将新信息添加到上下文中，循环重复直至任务完成

实例：用户问"我的订单号12345在哪？" → Agent制定计划（识别订单→查询物流→反馈结果）→ 调用find_order工具→ 获取追踪号→ 调用get_shipping_status→ 返回"您的订单正在配送中"

## 1.5 工具类型与功能

工具是连接Agent推理与现实世界的桥梁，使其能够突破静态训练数据的局限。

### 信息检索类工具

- 检索增强生成（RAG）：给Agent一张"图书馆卡"查询外部知识库（向量数据库或知识图谱）

- 自然语言到SQL（NL2SQL）：允许Agent查询数据库回答分析问题

- Google Search集成：通过Gemini模型直接调用网络搜索获取实时信息

### 行动执行类工具

- API调用：发送邮件、安排会议、更新CRM记录等

- 代码执行：在安全沙箱中生成并执行SQL查询或Python脚本

- 人类在环中（HITL）：暂停工作流请求确认或获取用户输入

### 函数调用标准

- OpenAPI规范：结构化合同，描述工具目的、参数和预期响应

- MCP（Model Context Protocol）：开放标准，便于工具发现和连接

- 本地工具：如Gemini内置Google Search，函数调用作为LM调用的一部分

## 1.6 协调层设计

协调层是连接模型和工具的中枢神经系统，是运行"思考、行动、观察"循环的引擎。

### ReAct框架（Reason + Action）

最常用的协调模式，将推理和行动交错执行：

- Reason：Agent评估目标和当前状态，形成关于下一步的假设

- Act：Agent选择并调用相应工具

- Observe：Agent接收工具输出，将新信息整合到上下文中

- 循环重复直至目标达成

### 演进路径

- 基础RAG：通过语义相似性检索文本，将相关信息作为上下文传递给LLM。适合直接问答，但对复杂关系查询表现不足

- GraphRAG：构建知识图谱，不仅匹配相似短语，还理解概念之间的关系。例如医疗AI需理解"症状→病因→治疗"的关系链

- Agentic RAG：Agent不再是被动接收检索结果，而是主动推理参与者。能分析复杂查询、制定多步计划、执行多次工具调用以找到最佳信息

## 2.5 确保Agent可靠性和责任性

从传统确定性软件到随机代理系统的转变需要新的操作哲学。

### AgentOps：生产就绪Agent框架

- 像A/B实验一样仪表成功：定义KPIs（目标完成率、用户满意度、任务延迟、操作成本）

- 质量而非通过/失败：使用"LM作为评判"评估质量，根据预定义标准评估Agent输出

- 指标驱动开发：将新版本应用于完整评估数据集，直接对比评分与现有生产版本

- 使用OpenTelemetry跟踪调试：记录Agent完整执行路径，诊断和修复根本原因

- 珍惜人类反馈：用户错误报告和"thumbs down"是改进Agent的最有价值资源

## 2.6 安全与责任AI

### 保护单个Agent：信任权衡

- 权限最小化：Agent应仅拥有执行任务所需的最小权限

- 输入验证：验证所有外部输入，防范提示注入攻击

- 输出过滤：确保Agent输出符合安全策略和内容标准

- 审计日志：记录所有Agent操作和决策路径

### 多Agent安全

- Agent间认证：使用A2A协议确保Agent间通信的安全性和身份验证

- 任务隔离：不同Agent的任务执行环境应相互隔离

- 权限委派控制：明确定义Agent可以委派给其他Agent的权限范围
