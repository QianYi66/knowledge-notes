🧭 如何成为AI原生工程师？
这篇文章讨论了 AI 时代工程师的职业变化，并提出三个关键判断：
- 初级工程师正面临史无前例的就业寒冬。
- 前 1% 顶尖工程师正在构建 Agent 友好型代码库。
- 新人比资深工程师更适合 AI 创业。
📉 初级工程师就业困难的三重原因
文章指出，初级工程师的困境不是个人问题，而是三股力量叠加造成的：
- 疫情后裁员潮：企业发现裁掉 20–30% 人员仍能正常运转。
- CS 毕业生数量暴涨：供给远超以往。
- AI 技术成熟：企业不再问“你会不会写代码”，而是问“你是不是 AI 原生”。
因此，新一代工程师必须同时具备 扎实基础 + AI 原生能力。
🧠 什么是 AI 原生工程师？
Mihel 给出的定义包括：
- 传统能力：系统设计、算法思维、扎实编程基础。
- 新时代能力：熟练使用 Agent 工作流、理解多 Agent 协作、具备上下文切换能力。
他强调： 不要一上来就用 10 个 Agent，从一个 Agent 打磨到极致才是关键。
🧩 Agent 友好型代码库：未来工程师的核心竞争力
Mihel 提出一个关键概念：Agent-friendly codebase。
一个代码库是否适合 Agent 工作，取决于：
- 测试覆盖率是否足够（测试是 Agent 的“契约”）。
- README 与代码是否一致，避免 Agent 困惑。
- 代码结构是否清晰、有 lint 规则。
- 第一版代码是否稳固、严密，避免 Agent 放大错误。
他指出： Agent 会快速放大错误，因此第一版代码必须非常稳固。
🎨 打造卓越软件的两个关键：品味 + 实验精神
Mihel 认为优秀软件的差异来自两个维度：
- 品味（Taste）：愿意在完成需求后继续打磨、增强健壮性、构建更复杂系统。
- 实验精神（Experimentation）：持续重写、持续迭代、持续验证，例如 Claude 团队每 1–2 周用 Claude 重写一次 Claude。
🚀 为什么新人更适合 AI 创业？
Mihel 的观点非常反直觉：
- 新人像海绵，更开放、更敢试。
- 他们不会被“行业有多难”吓倒。
- 更快掌握 AI 原生技能。
- 在某些方面甚至能超越资深开发者。
他认为： AI 原生时代，新人反而比资深工程师更具创业优势。
📌 总结
这篇文章传递的核心信息是：
- AI 正在重塑工程师的能力模型。
- 未来的竞争力不再是“写代码”，而是“构建 Agent 能理解并安全扩展的系统”。
- 新人如果掌握 AI 原生技能，反而更有机会。
